# 目的

ユーザの状態や製品知識を理解して接客を行うAIエージェントのデモを作成したい

# 要件

- AIエージェントは顧客状態や製品知識の入ったグラフDBを参照してユーザへの接客を行う
- グラフDBはデータ基盤内のテーブルから作る

# 背景知識

- 想定事業はヘッドホン事業で、複数種類のイヤホンとヘッドホンを扱っている
- 製品と一緒に利用するアプリもあり、これによって各種設定が可能
- テーブルには以下のようなものがある
  - fact_user_product: ユーザの製品所有状況に関するテーブル
  - fact_app_use: dailyのアプリ利用状況に関するテーブル。どのユーザがどの機種でアプリをBG/FG利用したかがある
  - fact_battery_condition: 各機種のバッテリー状況に関するテーブル
  - dim_user_profile: ユーザのデモグラフィック情報
  - static_description_model: 各機種のスペックや価格などの説明があるテーブル

# 作業依頼内容

1. テーブルスキーマおよびその値を現実的なものになるように生成する。テーブルは上にある例以外にも現実にありそうなものを作っていい。
2. テーブルからグラフDBを作成する
3. ユーザIDおよび質問(例: 次どんな機種買えばいい？)を入力すると、グラフDBを参照して適切にLLMが回答してくれるデモアプリを作成する。

# 注意点

- 適切にgit commit pushする
- Pythonで実装。uvで管理。
- グラフDBやエージェントオーケストレーションの仕組みはpythonと相性が良く、かつモダンなものを使用すること
- LLM APIを利用していい。.envに以下の形式で置いておく
  OPENAI_API_KEY=
  ANTHROPIC_API_KEY=
  GOOGLE_API_KEY=
- フォールバックは実装しない。
- dcos/research-result.mdにグラフを元にした接客エージェントに関する話があるので、それを参考にして設計してもいい。
